  • FiveDayChamp

    • What does it do?

    This is a training tool for future Jeopardy champions.

    Players set their difficulty level and categories, then the app serves up practice questions that match their preferences. Players log if their response was correct or incorrect and FiveDayChamp tracks their progress over time.

    How does it work?

    A node.js server is connected to a MongoDB cluster containing collections for users, questions, and responses. The server creates queries to the db based on the user's difficulty and category preferences, then delivers questions to the player in the response. When a user indicates if they've gotten the question correct or not, the response is saved in the db and used to track their stats.

    Stack

    Node.js, MongoDB, Express, JavaScript, HTML/CSS, and OAuth.

  • DrinkWa.org

    • What does it do?

    Gives users a sneak-peek at upcoming beers from Washington brewers.

    The latest beers from all breweries in the state are shown by default. Users can use the search bar to select specific beers or breweries. They can also log-in to leave ratings for the artwork of the new label.

    How does it work?

    When a brewery wants to release a new beer, they must first get government approval for the label that will be on it before it can be released. This approval process results in public records, which are generally searchable before the beer is available in the market. The data collection component of this project uses web scraping to search these public records for beers from breweries located in the state of Washington and node.js site to display them in a user-friendly and searchable way.

    Stack

    AWS S3, Python, Node.js, MongoDB, Express, JavaScript, HTML/CSS, OAuth.
